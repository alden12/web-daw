UX Design · Concept

web-daw on touch — phone & tablet

The desktop is a four-region grid you see all at once. On touch, that becomes one focused region at a time, navigated from a thumb-reachable tab bar, with the transport always present and the AI as a first-class way to make the edits that fingers are bad at. Same stores, same two-voice colour — a new shell, not a new app.

Swap the shell

NAVIGATE

Tabs + sheets

Bottom tab bar (thumb zone), persistent transport up top, slide-up sheets for transient tasks.

INPUT

Pointer-events touch

One drag path for mouse/touch/Pencil. Pinch-zoom, explicit draw/select/erase tools, big hit targets.

CREATE

Agent-forward

Describing beats poking. On mobile the agent is the primary way to build; touch is for fine-tuning.

Landscape earns two panels at once

  1. 1 Master/detail: rail + track list beside the selected track's workbench — closer to desktop.
  2. 2 The device chain (instrument → effects) is horizontal, knobs enlarged for touch / Pencil.
  3. 3 Roll + a slim timeline strip coexist; transport & the same tab set stay put.
  4. 4 Detect a stylus/mouse via pointerType and give it the precise interactions.
Packaging path: ship a PWA first (installable, offline, free — and it tells us whether an OS webview can carry real-time AudioWorklet audio, the one real risk), then Tauri v2 for the app stores — one project covers desktop and mobile. Capacitor is the fallback if Tauri-mobile plugins fall short.

Concept mockup — layout & interaction model only, not a working build. Tokens and two-voice colour pulled from src/index.css; full write-up in docs/DESIGN.md (Mobile / responsive layout).
